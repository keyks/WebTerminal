## 1. 测试范围与目标

### 1.1 测试范围

| 模块 | 核心功能 | 测试优先级 |
|------|----------|-----------|
| 认证系统 | JWT 登录/登出、Token 刷新、登录限流 | P0 |
| SSH 连接管理 | 连接创建/断开、密码/私钥认证、连接池 | P0 |
| 终端模拟 | WebSocket 通信、输入/输出、终端缩放 | P0 |
| 数据库层 | CRUD、加解密、迁移兼容 | P0 |
| SFTP 文件管理 | 目录浏览、文件读写、上传下载、权限修改 | P1 |
| 安全机制 | 命令拦截、路径遍历防护、审计日志 | P1 |
| 系统监控 | CPU/内存/磁盘/网络数据采集 | P1 |
| 告警引擎 | 规则管理、告警触发、冷却机制 | P2 |
| 集群运维 | 批量命令、文件分发、巡检报告 | P2 |
| AI 功能 | 诊断、对话、命令分析、知识库 | P2 |
| 会话管理 | 会话录制、恢复、空闲超时 | P2 |

### 1.2 测试目标

- **功能性**：所有核心 API 端点返回正确结果
- **稳定性**：72 小时持续运行无内存泄漏、无连接泄漏
- **性能**：100 并发 SSH 会话下终端延迟 < 200ms
- **安全性**：通过 OWASP Top 10 基础检查
- **兼容性**：Python 3.9-3.12，主流浏览器（Chrome/Firefox/Edge）

### 2.1 硬件建议

| 项目 | 最低配置 | 推荐配置 |

### 2.2 软件依赖

```bash
# Python 环境
Python >= 3.9

# 安装测试依赖
cd backend
pip install -r requirements.txt
pip install pytest pytest-cov pytest-timeout pytest-xdist locust

# 目标 SSH 服务器（用于集成测试）
# 需要一台运行 SSH 服务的 Linux 主机

  -p 2222:22 \
  -e SSH_USERNAME=testuser \
  -e SSH_PASSWORD=testpass \
  linuxserver/openssh-server
```

## 10. 测试执行脚本

### 10.1 一键运行所有测试

```bash
# 运行单元测试
cd d:/WebShell
python -m pytest tests/ -v --tb=short -x

# 运行单元测试 + 覆盖率
python -m pytest tests/ -v --cov=backend --cov-report=html --cov-report=term

# 只跑单元测试（跳过集成和稳定性测试）
python -m pytest tests/test_unit/ -v

# 只跑集成测试
python -m pytest tests/test_integration/ -v

# 只跑稳定性测试（长时间运行）
python -m pytest tests/test_stability.py -v --timeout=3600

# 并发运行（加速）
python -m pytest tests/ -n auto -v
```

```bash
# 安装 locust
```
